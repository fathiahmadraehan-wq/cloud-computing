Bagian 01

Pengertian Cloud Computing

Cloud computing adalah model pengiriman layanan komputasi melalui internet yang memungkinkan akses on-demand ke sumber daya seperti server, storage, database, jaringan, dan perangkat lunak.

NIST · Karakteristik 1

🌐 On-Demand Self-Service

Pengguna dapat memperoleh kapasitas komputasi secara otomatis tanpa interaksi langsung dengan penyedia.

NIST · Karakteristik 2

🔗 Broad Network Access

Layanan tersedia melalui jaringan dan dapat diakses via smartphone, tablet, laptop, maupun workstation.

NIST · Karakteristik 3

🏊 Resource Pooling

Sumber daya dikumpulkan untuk melayani banyak konsumen dengan model multi-tenant yang dinamis.

NIST · Karakteristik 4

⚡ Rapid Elasticity

Kapasitas dapat diprovisikan dan dirilis secara elastis sesuai permintaan.

NIST · Karakteristik 5

📊 Measured Service

Penggunaan sumber daya diukur secara otomatis — transparan untuk penyedia dan konsumen (pay-as-you-go).

Model Layanan Cloud

Tiga model layanan utama yang mendefinisikan apa yang dikelola penyedia vs. konsumen.

AspekIaaSPaaSSaaS
DefinisiInfrastruktur virtual: server, jaringan, storagePlatform pengembangan + runtimeAplikasi siap pakai via browser
Dikelola PenggunaOS, middleware, app, dataAplikasi dan dataHanya konfigurasi pengguna
Fleksibilitas⭐⭐⭐⭐⭐ Sangat tinggi⭐⭐⭐⭐ Tinggi⭐⭐ Terbatas
ContohAWS EC2, Azure VMHeroku, Google App EngineGmail, Microsoft 365

Bagian 03

Model Penerapan Cloud

Model penerapan menentukan di mana infrastruktur cloud dikelola dan siapa yang memiliki akses.

🌐

Public Cloud

Infrastruktur dimiliki penyedia pihak ketiga. Contoh: AWS, GCP, Azure.

✓ Biaya rendah   ✓ Skalabilitas tinggi
🏢

Private Cloud

Digunakan eksklusif satu organisasi. Kontrol dan keamanan penuh.

✓ Keamanan tinggi   ✓ Kontrol penuh
🔀

Hybrid Cloud

Kombinasi public dan private cloud. Data berpindah fleksibel.

✓ Fleksibel   ✓ Hemat biaya
🕸️

Multi-Cloud

Menggunakan layanan dari dua penyedia atau lebih. Hindari vendor lock-in.

✓ Bebas vendor lock-in   ✓ Redundansi

Keamanan Cloud & Tata Kelola

Keamanan cloud adalah tanggung jawab bersama (shared responsibility) antara penyedia dan pengguna cloud.

☁️ Penyedia Bertanggung Jawab

  • ✓ Keamanan fisik data center
  • ✓ Keamanan jaringan
  • ✓ Keamanan hypervisor
  • ✓ Hardware dan fasilitas

👤 Pengguna Bertanggung Jawab

  • ✓ Enkripsi data dan aplikasi
  • ✓ Manajemen identitas (IAM)
  • ✓ Konfigurasi firewall
  • ✓ Kepatuhan regulasi data
