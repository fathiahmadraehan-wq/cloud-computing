Bagian 01

Pengertian Cloud Computing

Cloud computing adalah model pengiriman layanan komputasi melalui internet yang memungkinkan akses on-demand ke sumber daya seperti server, storage, database, jaringan, dan perangkat lunak dengan model pembayaran pay-as-you-go.

NIST · Karakteristik 1

🌐 On-Demand Self-Service

Pengguna dapat memperoleh kapasitas komputasi secara otomatis tanpa interaksi langsung dengan penyedia layanan.

NIST · Karakteristik 2

🔗 Broad Network Access

Layanan tersedia melalui jaringan dan dapat diakses via smartphone, tablet, laptop, maupun workstation.

NIST · Karakteristik 3

🏊 Resource Pooling

Sumber daya dikumpulkan untuk melayani banyak konsumen dengan model multi-tenant yang dinamis.

NIST · Karakteristik 4

⚡ Rapid Elasticity

Kapasitas dapat diprovisikan dan dirilis secara elastis sesuai permintaan, tampak tidak terbatas bagi konsumen.

NIST · Karakteristik 5

📊 Measured Service

Penggunaan sumber daya diukur secara otomatis — transparan untuk penyedia dan konsumen (pay-as-you-go).

Bagian 03

Model Penerapan Cloud

Model penerapan menentukan di mana infrastruktur cloud dikelola dan siapa yang memiliki akses.

🌐

Public Cloud

Infrastruktur dimiliki penyedia pihak ketiga. Dibagikan antar banyak organisasi. Contoh: AWS, GCP, Azure.

✓ Biaya rendah   ✓ Skalabilitas tinggi   ✓ Tanpa maintenance
🏢

Private Cloud

Digunakan eksklusif satu organisasi. Kontrol dan keamanan penuh. Cocok untuk bank dan pemerintah.

✓ Keamanan tinggi   ✓ Kontrol penuh   ✓ Kustomisasi
🔀

Hybrid Cloud

Kombinasi public dan private cloud. Data dan aplikasi berpindah fleksibel sesuai kebutuhan.

✓ Fleksibel   ✓ Hemat biaya   ✓ Kepatuhan data
🕸️

Multi-Cloud

Menggunakan layanan dari dua penyedia atau lebih. Menghindari vendor lock-in dan memaksimalkan layanan terbaik.

✓ Bebas vendor lock-in   ✓ Redundansi

Bagian 04

Teknologi Enabler: Virtualisasi

Virtualisasi memungkinkan satu server fisik menjalankan banyak mesin virtual sekaligus, fondasi utama cloud computing.

🖥️ Tradisional
📦 Aplikasi
🪟 Sistem Operasi
⚙️ Hardware Fisik
1 Server = 1 Aplikasi
☁️ Virtualisasi
📦 App A
OS Linux
📦 App B
OS Windows
🔄 Hypervisor (VMware/KVM)
⚙️ Hardware Fisik (1 Server)
🖥️

Type 1 Hypervisor

Bare-metal, langsung di atas hardware. Contoh: VMware ESXi, KVM, Hyper-V. Performa & keamanan tinggi.

💻

Type 2 Hypervisor

Di atas OS host. Mudah diinstal, cocok pengembangan. Contoh: VirtualBox, VMware Workstation.

🐳

Containerization

Virtualisasi ringan berbagi kernel OS. Docker & Kubernetes — lebih efisien dan cepat di-deploy.

Bagian 06

Keamanan Cloud & Tata Kelola

Keamanan cloud adalah tanggung jawab bersama (shared responsibility) antara penyedia dan pengguna cloud.

☁️ Penyedia Bertanggung Jawab

  • ✓ Keamanan fisik data center
  • ✓ Keamanan jaringan infrastruktur
  • ✓ Keamanan hypervisor
  • ✓ Hardware dan fasilitas fisik

👤 Pengguna Bertanggung Jawab

  • ✓ Enkripsi data dan aplikasi
  • ✓ Manajemen identitas (IAM)
  • ✓ Konfigurasi firewall & akses
  • ✓ Kepatuhan regulasi data
