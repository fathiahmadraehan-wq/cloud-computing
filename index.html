Bagian 01

Pengertian Cloud Computing

Cloud computing (komputasi awan) adalah model pengiriman layanan komputasi — termasuk server, penyimpanan, basis data, jaringan, perangkat lunak, analitik, dan kecerdasan buatan — melalui internet ("awan") untuk menawarkan inovasi yang lebih cepat, sumber daya yang fleksibel, dan skala ekonomi.

🌐
Definisi NIST

On-Demand Self-Service

Pengguna dapat memperoleh kapasitas komputasi secara otomatis sesuai kebutuhan tanpa memerlukan interaksi langsung dengan penyedia layanan.

🔗
Aksesibilitas

Broad Network Access

Kemampuan tersedia melalui jaringan dan dapat diakses melalui berbagai perangkat klien seperti smartphone, tablet, laptop, dan workstation.

🏊
Efisiensi

Resource Pooling

Sumber daya komputasi penyedia dikumpulkan untuk melayani banyak konsumen menggunakan model multi-penyewa (multi-tenant) yang dinamis.

⚡
Elastisitas

Rapid Elasticity

Kapasitas dapat diprovisikan dan dirilis secara elastis — bahkan otomatis — sesuai permintaan, tampak tidak terbatas bagi konsumen.

📊
Akuntabilitas

Measured Service

Sistem cloud secara otomatis mengontrol dan mengoptimalkan penggunaan sumber daya dengan kemampuan pengukuran yang transparan (pay-as-you-go).

Bagian 03

Model Penerapan Cloud

Model penerapan menentukan di mana dan bagaimana infrastruktur cloud dikelola serta siapa yang memiliki akses ke sumber daya tersebut.

🌐

Public Cloud

Infrastruktur dimiliki dan dioperasikan oleh penyedia cloud pihak ketiga. Sumber daya seperti server dan storage dibagikan antar banyak organisasi. Contoh: AWS, Google Cloud, Azure.

✓ Biaya rendah   ✓ Skalabilitas tinggi   ✓ Tanpa maintenance
🏢

Private Cloud

Infrastruktur cloud yang digunakan secara eksklusif oleh satu organisasi. Dapat di-host secara on-premise atau oleh pihak ketiga, memberikan kontrol dan keamanan penuh.

✓ Keamanan tinggi   ✓ Kontrol penuh   ✓ Kustomisasi
🔀

Hybrid Cloud

Kombinasi public dan private cloud yang memungkinkan data dan aplikasi berpindah antara keduanya. Memberikan fleksibilitas dan lebih banyak opsi penerapan.

✓ Fleksibel   ✓ Hemat biaya   ✓ Kepatuhan data
🕸️

Multi-Cloud

Penggunaan layanan cloud dari dua atau lebih penyedia cloud yang berbeda. Menghindari vendor lock-in dan memanfaatkan layanan terbaik dari masing-masing penyedia.

✓ Bebas vendor lock-in   ✓ Redundansi   ✓ Best-of-breed

Bagian 04

Teknologi Enabler: Virtualisasi

Virtualisasi adalah fondasi utama cloud computing. Teknologi ini memungkinkan satu server fisik menjalankan banyak mesin virtual (VM) secara bersamaan, memaksimalkan utilisasi hardware dan efisiensi biaya.

🪟 Sistem Operasi
⚙️ Hardware Fisik
1 Server = 1 Aplikasi
☁️ Arsitektur Virtual
📦 App A
OS Linux
📦 App B
OS Windows
🔄 Hypervisor (VMware / KVM / Hyper-V)
⚙️ Hardware Fisik (1 Server)
🖥️

Type 1 Hypervisor (Bare-Metal)

Berjalan langsung di atas hardware fisik tanpa OS host. Performa lebih tinggi dan aman. Contoh: VMware ESXi, Microsoft Hyper-V, KVM, Xen.

💻

Type 2 Hypervisor (Hosted)

Berjalan di atas OS host yang sudah ada. Lebih mudah diinstal, cocok untuk pengembangan. Contoh: VMware Workstation, VirtualBox, Parallels.

🐳

Containerization

Alternatif virtualisasi yang lebih ringan menggunakan kernel OS bersama. Docker dan Kubernetes adalah teknologi utama. Lebih efisien dan cepat di-deploy.

Manfaat VirtualisasiPenjelasanDampak
🔋 Efisiensi Resource Utilisasi server meningkat dari rata-rata 10–15% menjadi 70–80% Penghematan biaya hardware hingga 70%
⚡ Provisioning Cepat VM baru dapat dibuat dalam hitungan menit vs. minggu untuk server fisik Agility bisnis meningkat drastis
🛡️ Disaster Recovery Snapshot dan migrasi VM memudahkan backup dan pemulihan RTO/RPO lebih rendah
🌿 Green IT Lebih sedikit server fisik berarti konsumsi daya dan pendingin berkurang Jejak karbon data center berkurang
Bagian 05

Penyimpanan Data dan Jaringan di Cloud

Cloud menyediakan berbagai jenis penyimpanan data dan infrastruktur jaringan yang mendukung ketersediaan, durabilitas, dan performa layanan secara global.

🗄️ Block Storage

Penyimpanan tingkat blok untuk performa I/O tinggi

✓Cocok untuk database dan OS
✓Latensi sangat rendah (sub-ms)
✓Performa konsisten dan prediktif
✓Contoh: AWS EBS, Azure Disk, GCP Persistent Disk

📦 Object Storage

Penyimpanan berbasis objek untuk skala masif

✓Skalabilitas hampir tak terbatas
✓Durabilitas 11 sembilan (99.999999999%)
✓Cocok untuk media, backup, data lake
✓Contoh: AWS S3, Google Cloud Storage, Azure Blob

📁 File Storage

Sistem file terkelola berbasis NFS/SMB

✓Akses file sharing antar banyak VM
✓Hierarki direktori familiar
✓Integrasi mudah dengan aplikasi legacy
✓Contoh: AWS EFS, Azure Files, GCP Filestore

🌐 Cloud Networking

Jaringan virtual dan konektivitas global

✓VPC (Virtual Private Cloud) — jaringan terisolasi
✓Load Balancer — distribusi trafik otomatis
✓CDN — konten dikache dekat pengguna
✓VPN & Direct Connect — koneksi privat aman

Bagian 06

Keamanan Cloud & Tata Kelola

Keamanan cloud merupakan tanggung jawab bersama (shared responsibility) antara penyedia cloud dan pengguna. Pemahaman model ini penting untuk menghindari celah keamanan.

🤝 Model Shared Responsibility

☁️ Penyedia Cloud Bertanggung Jawab

  • ✓ Keamanan fisik data center
  • ✓ Keamanan jaringan infrastruktur
  • ✓ Keamanan hypervisor
  • ✓ Hardware dan fasilitas
  • ✓ Software managed services

👤 Pelanggan Bertanggung Jawab

  • ✓ Keamanan data dan enkripsi
  • ✓ Manajemen identitas (IAM)
  • ✓ Konfigurasi aplikasi
  • ✓ Kebijakan akses & firewall
  • ✓ Kepatuhan regulasi data
🔐

Identity & Access Management

Kontrol siapa yang dapat mengakses apa, kapan, dan bagaimana. MFA, RBAC, least privilege.

🔒

Enkripsi Data

Data dienkripsi saat diam (at-rest) dan dalam perjalanan (in-transit) menggunakan AES-256 dan TLS.

🛡️

Firewall & WAF

Web Application Firewall melindungi dari serangan SQL injection, XSS, DDoS, dan ancaman web umum.

👁️

Monitoring & Audit

Log aktivitas, SIEM, dan peringatan real-time membantu mendeteksi dan merespons ancaman lebih cepat.

📋

Compliance & Governance

Memastikan kepatuhan terhadap regulasi: ISO 27001, SOC 2, GDPR, PCI-DSS, dan peraturan lokal.

🔄

Disaster Recovery

Strategi backup, replikasi multi-region, dan rencana pemulihan bencana untuk ketersediaan tinggi.

AncamanDeskripsiMitigasi
🚨 Data BreachKebocoran data akibat konfigurasi keliru atau eksploitasiEnkripsi, IAM ketat, audit berkala
🔑 Account HijackingPengambilalihan akun via phishing atau credential stuffingMFA, monitoring login anomali
🌊 DDoS AttackBanjir trafik untuk melumpuhkan layananAnti-DDoS, rate limiting, CDN
💉 Injection AttackSQL/XSS injection melalui input tidak tervalidasiWAF, input validation, prepared statements
🔓 MisconfigurationBucket S3 publik, port terbuka, hak akses berlebihCSPM tools, infrastructure as code, review

⚙️
Microservices

Arsitektur Microservices

Aplikasi dipecah menjadi layanan kecil independen yang berkomunikasi via API. Setiap service dapat di-deploy, di-scale, dan diperbarui secara terpisah.

🐳
Kubernetes

Container Orchestration

Kubernetes (K8s) mengelola deployment, scaling, dan load balancing container secara otomatis. Standar industri untuk cloud-native workload.

⚡
Serverless

Serverless Computing

Developer menulis fungsi tanpa mengelola server. Penyedia cloud mengelola infrastruktur. Bayar hanya saat fungsi dieksekusi (event-driven).

🔄
DevOps

CI/CD & DevOps

Continuous Integration/Continuous Deployment mengotomatisasi build, test, dan release. Mempercepat siklus pengembangan dan mengurangi error manual.
